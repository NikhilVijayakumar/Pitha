# STD-010: Philosophy Checks Use Inverted Logic

**Domain:** philosophy
**Severity:** High
**Impact:** Artifically deflates audit score from ~95+ to 84.21
**Status:** Open

## Description

Philosophy audit checks appear to test for the *absence* of sections rather than their *presence*. The philosophy document has all 3 required sections (guiding_principles, values, tradeoffs) plus optional sections (purpose, traceability), but checks report them as errors.

## Affected Checks

- `phil-001` — Principles documented (reports Error when Principles section EXISTS)
- `phil-002` — Values documented (reports Error when Values section EXISTS)
- `phil-003` — Trade-offs documented (reports Error when Trade-offs section EXISTS)

## Evidence

Querying the knowledge DB confirms all required sections exist with correct semantic types:

- `guiding_principles`: 1 section (Principles) — contains 6 named principles
- `values`: 1 section (Values) — contains 5 named values
- `tradeoffs`: 1 section (Trade-offs) — contains 5 named trade-offs
- `purpose`: 1 section (Purpose)

All sections have correct content, proper structure, and appropriate length.

## Root Cause

Check definitions in `audit/deterministic/document/02-philosophy.yaml` likely use inverted condition logic (e.g., `not_exists` instead of `exists`) or test for the wrong semantic type.

## Recommended Fix

Verify check definitions test `exists: true` for the required semantic types (`guiding_principles`, `values`, `tradeoffs`) and correct any inverted conditions.
