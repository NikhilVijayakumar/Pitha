# STD-009: Architecture Collection-Level Checks Fire on Individual Documents

**Domain:** architecture
**Severity:** High
**Impact:** Artifically deflates audit score from ~95+ to 87.91
**Status:** Open

## Description

Architecture audit checks (e.g., `arch-sec-guiding-001`, `arch-sec-build-001`, `arch-sec-rationale-001`) are defined at document scope but evaluate collection-level requirements. Every check fires against all 9 architecture documents instead of evaluating the collection as a whole.

## Affected Checks

All `arch-sec-*` section checks fire 9× (once per document) instead of 1× (once per collection):

- `arch-sec-guiding-001/002` — Guiding Principles
- `arch-sec-build-001/002/005` — Build Standards
- `arch-sec-rationale-001` — Rationale
- `arch-sec-code-001/002` — Code Standards
- `arch-sec-constraints-003` — Constraints
- `arch-sec-hook-001/002/003` — Hook Conventions
- `arch-sec-pat-001/002/005` — Component Patterns
- `arch-sec-state-001/002/004/005` — State Management
- `arch-sec-trace-002/003` — Traceability

## Root Cause

Check definitions in `audit/deterministic/section/05-architecture/*.yaml` use document scope when they should use collection scope. The standard itself states: "Architecture quality is evaluated across the complete documentation collection."

## Evidence

All 9 architecture documents have the 5 required sections (system_overview, component_model, communication_paths, data_flow, security_considerations) plus optional sections (purpose, rationale, constraints, traceability). Content quality is high. The 87.91 score is inflated findings, not missing content.

## Recommended Fix

Update check definitions to use collection scope, or add a `scope: collection` flag to section-level checks that represent collection-wide requirements.
